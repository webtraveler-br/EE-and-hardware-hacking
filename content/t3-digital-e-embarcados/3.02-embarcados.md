Manipulação de bits: set/clear/toggle
@@
Set: REG |= (1 << n). Clear: REG &= ~(1 << n). Toggle: REG ^= (1 << n)

@card dV8xcFpLYnVnPA
SPI modos (CPOL/CPHA)
@@
4 modos: CPOL define polaridade idle do clock, CPHA define borda de amostragem. Modo 0 (0,0) é o mais comum

@card Z0QyNSlfV1lNVQ
I2C: por que pull-up é obrigatório
@@
Barramento é open-drain — dispositivos só puxam para GND, resistores puxam para VCC (tipicamente 4.7 kΩ)

@card ZEJ2I19PMktRIQ
Cálculo de baud rate (UART)
@@
UBRR = $\frac{f_{CLK}}{16 \cdot baud} - 1$. Erro > 2% causa perda de dados

@card blFBX3NFbFQ4eA
Prioridade em RTOS
@@
Tarefas com prioridade maior preemptam as de menor. Inversão de prioridade: baixa segura mutex que alta precisa → deadlock

@card Z3dLXWhHQEhvaQ
CAN bus — características
@@
Barramento multi-mestre, diferencial (CANH/CANL), até 1 Mbit/s 40m. Arbitragem por prioridade de ID sem colisão. Dominante em automotivo e industrial.

@card cjJKU1ZsY0spUQ
RS-485 — características
@@
Serial diferencial half-duplex, até 10 Mbit/s em distâncias curtas, 1200m a 100 kbit/s. Até 32 dispositivos no barramento. Base física do Modbus RTU.

@card enE2O05hOl9ZSQ
Bare metal vs HAL vs RTOS — quando usar cada
@@
Bare metal: máxima performance, registradores diretos. HAL: portabilidade entre MCUs. RTOS: multitarefa com deadlines (motor + comunicação + display).

@card MEdaemxJPnNISw
Pinmux / Alternate Function em MCUs
@@
Cada pino pode ser GPIO, UART TX, SPI MOSI, PWM, ADC etc. Configurado via registradores de alternate function. Conflitos de pinmux = erro comum.

@card cFlyODwoWF5ncA
Current sensing com shunt + INA
@@
Resistor shunt (ex: 0.1Ω) em série com carga. INA219/226 mede tensão diferencial: $I = V_{shunt}/R_{shunt}$. Comunicação I2C com MCU.

@card LEpGTS0uMUIoRw
OTA (Over-The-Air) Update — conceito
@@
