# Roadmap do Deck 7.03-debug-e-firmware · Debug e Firmware

**Cards:** 32

## Missão do deck
Construir fluxo técnico de debug e engenharia reversa de firmware orientado a reprodutibilidade.

## Tópicos estudados
- JTAG: test access port (TAP), estado machine (TMS), data/instruction registers
- SWD (ARM): 2 fios (SWDIO, SWCLK), velocidade até 10MHz
- OpenOCD: framework open-source, suporte múltiplos processadores
- U-Boot bootloader: variáveis de ambiente, comandos (md, sf read, bootm)
- RDP levels (STM32): Level 0 (aberto), 1 (JTAG/SWD bloqueado, reversível), 2 (irreversível)
- ESP32 Secure Boot e Flash Encryption: bypass via análise de fuses
- Fault injection: voltage glitch, clock glitch, EMFI; momento crítico de exploração
- Firmware patching: modificação de binário, repack, recálculo de checksum
- Entropia: análise de compressão (gzip/LZMA) vs. criptografia
- Reverse engineering com Ghidra: strings, XREF, decompilation, finding vulnerabilities

## Sequência recomendada
1. Revisar os conceitos-base e resolver cards novos em blocos curtos de 20 a 30 minutos.
2. Resolver exercícios de aplicação imediata para transformar fórmula em tomada de decisão.
3. Executar os projetos propostos e registrar evidências no diretorio de projetos da trilha.
4. Fechar ciclo com revisão FSRS diária até estabilizar acurácia acima de 85%.

## Projetos conectados ao deck
1. **Debug Lab: OpenOCD + STM32 Blue Pill**
   - Escopo: Conecte via SWD em STM32 Blue Pill (~R$15), dump de flash completo, set breakpoints, modifique valores na RAM ao vivo. Portfolio: scripts OpenOCD, procedimento reproduzível e achados documentados.
   - Evidência: `content/roadmaps/projects/7.03-debug-e-firmware-01.md`
2. **Firmware Unpacker Pipeline**
   - Escopo: Script automatizado: input firmware.bin → binwalk extract → encontrar squashfs/cramfs → montar → extrair passwd/shadow/keys → gerar relatório. Portfolio: pipeline reproduzível e exemplos de saída.
   - Evidência: `content/roadmaps/projects/7.03-debug-e-firmware-02.md`
3. **CTF Caseiro de Reversing em Ghidra**
   - Escopo: Compile binário ARM com chave secreta plantada no código, reverta em Ghidra para encontrá-la. Escale dificuldade em 3 níveis (string literal → XOR → crypto). Portfolio: write-up estilo CTF para cada nível.
   - Evidência: `content/roadmaps/projects/7.03-debug-e-firmware-03.md`
4. **Voltage Glitch PoC com Arduino**
   - Escopo: Use Arduino extra para enviar glitches de tensão cronometrados e pular verificação de PIN em sketch de autenticação simples. Portfolio: setup documentado, taxa de sucesso e análise de timing.
   - Evidência: `content/roadmaps/projects/7.03-debug-e-firmware-04.md`

## Critério de conclusão
- Completar 100% dos cards do deck ao menos uma vez.
- Entregar pelo menos 3 dos 4 projetos com evidência técnica clara.
- Documentar erros recorrentes e ajustes no próprio roadmap do projeto.
